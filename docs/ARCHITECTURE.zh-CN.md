# MICP 架构

> 语言:[English](ARCHITECTURE.md) | **中文**

本文档描述 MICP 参考栈的内部结构及其设计取舍。

## 1. 设计目标

1. **可移植优先。** 纯 C11,无动态内存,不依赖 OS/传输 —— 同样适用于 MCU、RTOS 任务与
   宿主进程。
2. **传输无关。** 协议栈从不直接操作 socket 或 UART;它通过回调输出字节、通过 `feed()`
   消费字节。
3. **确定性。** 固定大小缓冲、每次调用工作量有界、显式的 tick 驱动定时器模型 —— 对硬实时
   调度友好。
4. **可测试、面向一致性。** 每一层独立单元测试,呼应 CANopen/EtherCAT 的一致性测试理念。

## 2. 分层结构

```
        ┌─────────────────────────────────────────────┐
        │ 应用层                                       │
        │  - 提供 output 回调(字节 -> 链路)          │
        │  - 提供 recv   回调(payload -> 应用)        │
        │  - 调用 send / connect / disconnect / tick   │
        └───────────────▲───────────────┬─────────────┘
                        │ on_recv        │ send/connect/tick
        ┌───────────────┴───────────────▼─────────────┐
        │ 会话层  (src/micp_session.c)                 │
        │  - 状态机 (DISCONN/CONNECTING/...)           │
        │  - 可靠 stop-and-wait + 重传定时器           │
        │  - 心跳 / 对端存活                            │
        │  - 字节流重组 (SOF 重同步)                   │
        └───────────────▲───────────────┬─────────────┘
                        │ decode         │ encode
        ┌───────────────┴───────────────▼─────────────┐
        │ 帧编解码  (src/micp_frame.c)                 │
        │  - 头 + payload 的序列化/反序列化            │
        │  - 长度与版本校验                            │
        └───────────────▲───────────────┬─────────────┘
                        │                │
        ┌───────────────┴───────────────▼─────────────┐
        │ 完整性  (src/micp_crc.c)                     │
        │  - CRC-16/CCITT-FALSE(按位、无查表)        │
        └──────────────────────────────────────────────┘
```

## 3. 模块

| 模块              | 文件                               | 职责                                            |
|-------------------|------------------------------------|-------------------------------------------------|
| 类型与错误码      | `micp_types.h` / `micp_types.c`    | 常量、枚举、错误码、名称辅助函数                |
| 完整性            | `micp_crc.{h,c}`                   | 一次性与增量 CRC-16                             |
| 帧编解码          | `micp_frame.{h,c}`                 | 单帧编/解码;边界与 CRC 校验                    |
| 会话              | `micp_session.{h,c}`               | 状态机、可靠性、成帧、定时器                    |
| 总入口            | `micp.h`                           | 单一 include + 库版本宏                          |

依赖图严格向下(session → frame → crc → types)。没有模块依赖更上层,这使编解码与 CRC
可独立复用(如用于嗅探器或一致性测试夹具)。

## 4. 数据流

### 发送
`micp_session_send()` 构造一个 `micp_frame_t`,调用 `micp_frame_encode()` 序列化(追加
CRC),并把字节交给应用的 `output` 回调。可靠发送时,编码后的字节缓存于 `pending_buf`,
以便重传路径无需重新编码即可重发。

### 接收
原始字节经 `micp_session_feed()` 进入,追加到每会话的重组缓冲(`rx_buf`)。解析器:
1. 丢弃任何前导非 `SOF` 字节以重同步;
2. 尝试 `micp_frame_decode()`;
3. 遇 `MICP_ERR_SHORT` 时等待更多字节;
4. 遇 `MICP_ERR_CRC` 或畸形头时丢弃一字节并重同步;
5. 成功则分派到 `handle_frame()`,并继续处理缓冲中剩余字节。

这使协议栈对任意切分/合并帧的字节流传输,以及注入杂散字节的线路噪声都具有鲁棒性。

## 5. 时序模型

协议栈**没有内部时钟**。应用通过周期性调用 `micp_session_tick(s, dt)`(定时器 ISR、
RTOS tick 钩子或轮询循环)推进时间。`dt` 单位由应用定义;所有超时
(`rto_ticks`、`heartbeat_ticks`、`peer_timeout_ticks`)使用同一单位。这让实现免于分配与
系统调用,并把分辨率/抖动的取舍交给集成方。

## 6. 内存模型

- 一个 `micp_session_t` 完全自包含(约 1.2 KB,主要由重组与重传缓冲构成,各为
  `MICP_MAX_FRAME` = 526 字节)。
- 库中任何位置都不使用堆。
- 一个会话代表一个 peer 关联。多个 peer = 多个会话。

## 7. 并发

单个会话**不**做内部同步。请在单一上下文中驱动每个会话,或用互斥锁包裹调用。不同会话
彼此独立,无需共享锁。

## 8. 可扩展性说明

- **滑动窗口**:当前可靠层为 stop-and-wait(仅一个在途帧)。帧已携带 16 位 `SEQ`,因此
  可在会话层加入窗口化 ARQ 而无需改动线序格式。
- **安全**:`FLAGS` 与版本化的头为认证/加密 profile 预留了空间(例如在 CRC 前追加一个
  AEAD 标签),可在未来 `VER` 中引入。
- **分片**:超过 512 字节的 payload 将在会话层分段;编解码与 CRC 不受影响。

## 9. 测试策略

| 用例集           | 覆盖内容                                                           |
|------------------|--------------------------------------------------------------------|
| `test_crc`       | CRC 已知答案、空输入、增量 == 一次性、敏感性                       |
| `test_frame`     | round-trip、零/最大 payload、CRC 错误、截断、坏 SOF、缓冲不足      |
| `test_session`   | 握手、可靠 ACK、busy、重传 + 耗尽、去重、字节分片、CRC 丢弃、       |
|                  | 垃圾重同步、断链、心跳、对端超时、send 需已连接、非 peer 帧拒绝     |
| `loopback_demo`  | 两个进程内节点的端到端正常路径                                     |

单元测试使用一个极小的纯头文件框架(`tests/micp_test.h`);进程非 0 退出码表示失败,
CTest 与 Makefile 都以此作为门禁。
